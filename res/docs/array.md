# Массивы

В Weasel массивы — это ссылочные структуры данных с фиксированным размером, которые хранят элементы одного типа. Благодаря строгой статической типизации, компилятор гарантирует, что в массив чисел никогда не попадет строка, а виртуальная машина обеспечит безопасность, перехватывая любые попытки выхода за границы памяти.

Все массивы в языке живут в динамической памяти (куче), а их жизненным циклом автоматически управляет встроенная система подсчета ссылок (ARC). Вам не нужно вручную освобождать память — виртуальная машина сделает это за вас, как только массив перестанет использоваться.

## Объявление и инициализация

Weasel предоставляет три способа создания массивов в зависимости от ваших задач: литералы, динамическое выделение памяти и вывод типа из контекста.

### 1. Литералы массивов

Если вы заранее знаете, какие элементы будут лежать в массиве, используйте синтаксис литералов `[ ]`. Компилятор сам вычислит тип массива на основе первого элемента и проверит остальные.

```wes
# Массив целых чисел
val array[int] numbers = [10, 20, 30]

# Массив строк
val array[string] words = ["Weasel", "Language", "Compiler"]

```

### 2. Динамическое выделение памяти

Когда размер массива вычисляется во время работы программы или элементы будут добавлены позже, память выделяется явно с указанием размера в круглых скобках `( )`. Все элементы при этом инициализируются нулями (или эквивалентом `false` / пустой ссылкой).

```wes
val uint dynamicSize = 100
var array[real] sensorData = array[real](dynamicSize)

```

### 3. Пустые массивы

Вы можете создать пустой массив, используя пустой литерал `[]`. Поскольку в пустом массиве нет элементов, компилятор Weasel требует явного указания типа переменной, чтобы понять, сколько памяти и для чего нужно выделить.

```wes
# Компилятор читает тип слева и применяет его к пустому литералу
val array[bool] flags = []

```

## Доступ к элементам и безопасность

Доступ к элементам массива осуществляется с помощью квадратных скобок `[ ]`. Индексация начинается с нуля.

Виртуальная машина Weasel аппаратно защищает вас от выхода за границы памяти (Out Of Bounds). Любая попытка прочитать или записать элемент по несуществующему индексу приведет к безопасной остановке программы с понятным сообщением об ошибке, а не к падению операционной системы.

```wes
var array[int] scores = [100, 95, 80]

# Чтение значения
print(scores[0])

# Запись нового значения
scores[1] = 99

# print(scores[5]) -> E_VM_OUT_OF_BOUNDS

```

## Изменчивость (Mutability)

Ключевые слова `val` и `var` при работе с массивами контролируют саму ссылку на массив, а не его содержимое.
Если вы объявили массив как `val`, вы не сможете присвоить этой переменной *другой* массив. Однако изменять элементы *внутри* этого массива — разрешено.

| Модификатор | Перезапись ссылки (`arr = newArr`) | Перезапись элементов (`arr[0] = 5`) |
| --- | --- | --- |
| `val` | Запрещена | Разрешена |
| `var` | Разрешена | Разрешена |

## Свойства массива

Массивы в Weasel оптимизированы для максимальной производительности. Для получения длины массива используется встроенное свойство `.size`.

Это свойство не является функцией и не требует накладных расходов на вызов. Виртуальная машина извлекает это значение за одну инструкцию напрямую из заголовка массива в оперативной памяти.

```wes
val array[int] data = array[int](42)

# Выведет 42
print(data.size)

```

## Многомерные массивы

Поскольку массив — это полноценный тип данных, вы можете создавать массивы массивов (матрицы) любой степени вложенности.

```wes
val array[array[int]] matrix = [
    [1, 2, 3],
    [4, 5, 6],
    [7, 8, 9]
]

# Доступ к центральному элементу (выведет 5)
print(matrix[1][1])

```
