## Правила языка, которые сэкономят вам часы (читать внимательно)

Weasel — статически типизированный, без неявных приведений. Подводные камни:

- **Дженериков нет.** Контейнер хранит только `int`. Тип элемента не параметризуется.
- **Метод класса** объявляется так: `ТипВозврата <- имя(параметры) { ... }`.
  Приватные хелперы — вне блока `public { }`, публичные — внутри него.
- **«Не равно»** — это оператор `><` (а не `!=`).
- **`array[int](n)`** создаёт массив на `n` элементов; индексация `a[i]`, длина `a.size`.
- **Границы массива не проверяются библиотекой** — это контракт вызывающего.
  В языке нет исключений; выход за границы аварийно останавливает программу.
- **Глобальных переменных нет** — `let/val/var` вне функции/класса не работают.
  Константы оформляйте нуль-арными функциями.
- **`when` пишется только многострочно**, тело ветви — сразу за `->` на той же строке.
- **Зарезервированные слова нельзя брать как имена**: `step, in, times, down, run,
  repeat, when, if, else, ...` (полный список — в документации). Особенно коварны
  `step`, `in`, `times`, `down`, `run`.
- Цикл по диапазону: `repeat (i in 0 .. n)` — верхняя граница **не включается**
  (это `[0, n)`).

## Часть 1 — Новый элемент стандартной библиотеки: `IntSet`

Реализуйте класс **`IntSet`** — множество целых чисел (без дубликатов) поверх
`array[int]` с линейным поиском. По стилю и устройству он должен быть «братом»
контейнеров из `stdlib/` (сравните с `stdlib/map.wes` — там та же идея с
`indexOf` и ростом ёмкости).

### Требуемый публичный API

| Метод | Тип | Поведение |
|---|---|---|
| `IntSet()` | конструктор | пустое множество (нач. ёмкость 4) |
| `IntSet(int initCapacity)` | конструктор | пустое множество с заданной ёмкостью (но не меньше 1) |
| `size()` | `int` | число элементов |
| `isEmpty()` | `bool` | `true`, если множество пусто |
| `contains(int value)` | `bool` | `true`, если значение есть |
| `add(int value)` | `void` | добавить значение; **дубликат игнорируется** |
| `remove(int value)` | `bool` | удалить значение; вернуть `true`, если было, иначе `false` |
| `get(int index)` | `int` | элемент по внутреннему индексу (для перебора `0..size()`) |

### Обязательные требования

- **Никаких дубликатов:** `add` существующего значения — no-op (размер не растёт).
- **Автоматический рост ёмкости** при заполнении (как в `stdlib/`: примерно ×1.5,
  с защитой от «застревания» на маленькой ёмкости).
- Приватный хелпер `indexOf(int value) -> int` (возвращает индекс или `-1`).
- Файл назвать **`set.wes`** и положить рядом с тестом: `res/lab/set.wes`
  (тогда `import "set"` найдёт его автоматически — см. `docs/modules.md`).

### Документация

Добавьте в свой отчёт фрагмент в стиле `docs/stdlib.md`: таблицу методов и
короткий пример использования `IntSet`.

## Как сдавать и как это проверяется

1. Положите `set.wes` в `res/lab/`.
2. Запустите приёмочный тест (из каталога `res`):

   ```
   wesc.exe lab/intset_test.wes
   ```

3. Вывод должен **посимвольно совпасть** с `lab/expected_output.txt`:

   ```
   true
   2
   true
   false
   false
   12
   true
   true
   false
   false
   11
   ```

   Проверка одной командой (PowerShell, из `res`):

   ```powershell
   wesc.exe lab/intset_test.wes | Out-String | %{ $_.Trim() } |
       Set-Content got.txt; Compare-Object (Get-Content got.txt) (Get-Content lab/expected_output.txt)
   ```

   Пустой результат `Compare-Object` = тест пройден.

**Сдать:** `set.wes`, отчёт с таблицей API и примером, вывод теста.

## Идеи для усложнённого варианта (по желанию / для сильных групп)

- `addAll(IntSet other)`, `union`, `intersect`, `difference` между двумя `IntSet`.
- `toArray() -> array[int]` (выгрузка элементов).
- Заменить линейный поиск на сортированное хранение + бинарный поиск в `contains`.
- Свой контейнер вместо `IntSet`: `IntDeque` (кольцевой буфер) или `MinHeap`
  (бинарная куча + heapsort, связка с Частью 0).
