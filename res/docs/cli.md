# Компилятор `wesc`: запуск, флаги и вывод ошибок

## Запуск

```
wesc <файл.wes> [флаги]
```

Пример (из каталога `res`):

```
wesc tests/modules/app.wes
```

По умолчанию `wesc` **компилирует и сразу исполняет** программу в виртуальной машине.

---

## Флаги

| Флаг | Значение |
|---|---|
| `-c`, `--check` | Только компиляция и проверка (типы, кодоген) — **без запуска**. Полезно для CI и `wkit build`. |
| `--stdlib <каталог>` | Каталог стандартной библиотеки для `import`. По умолчанию `stdlib`. |
| `-g <файл>` | Файл грамматики. По умолчанию `grammar.wesg`. |
| `--format` | Отформатировать исходный файл (режим форматтера). |
| `-l` | Включить подробный отладочный лог в консоль (токены, CST, AST, тайминги). |
| `-lf` | То же, но лог пишется в файл `log.txt`. |

---

## Вывод ошибок

**Ошибки компиляции всегда печатаются в `stderr`** — независимо от флагов. Не нужно включать `-l`/`-lf`, чтобы увидеть диагностику.

Флаги `-l` / `-lf` управляют **подробным отладочным логом** (визуализация токенов/дерева, тайминги) и не влияют на то, что ошибки видны в консоли. Дополнительно при `-lf` ошибки **дублируются в файл `log.txt`** вместе с отладочным логом (удобно для разбора постфактум). При `-l` ошибки не дублируются на консоль повторно — они и так уже в `stderr`.

Каждая ошибка содержит фазу компилятора, сообщение и — при наличии — ожидаемое/полученное значение и позицию (строка, столбец, файл):

```
[Semantic]  	Поле или метод не найдены для данного типа
		-> Строка 11, позиция 16 (tests/modules/app.wes)
```

Для нескольких файлов (через `import`) в позиции указывается, **в каком файле** ошибка — это важно, когда сущность объявлена в одном модуле, а используется в другом.

---

## Коды возврата

| Код | Значение |
|---|---|
| `0` | Успех (скомпилировано; при запуске — программа отработала). |
| ≠ `0` | Были ошибки компиляции (диагностика напечатана в `stderr`). |

Это позволяет надёжно использовать `wesc` в скриптах и в `wkit build`: проверяется код возврата, а текст ошибок виден пользователю.
