# Функции

Функции в Weasel — это именованные блоки кода с явно указанными типами параметров и возвращаемого значения. Компилятор строго проверяет типы на всех границах вызовов — неявных преобразований нет.

---

## Объявление

Функции объявляются на уровне файла. Синтаксис: сначала тип возвращаемого значения, затем стрелка `<-`, имя функции, список параметров в скобках и тело в фигурных скобках.

```wes
int <- add(int a, int b)
{
    return a + b
}

void <- greet(string name)
{
    println(name)
}
```

Функции, не возвращающие значения, объявляются с типом `void`. Такие функции могут содержать `return` без выражения или завершаться без явного `return`.

---

## Параметры — модификаторы передачи

Каждый параметр может иметь явный модификатор, управляющий тем, как значение передаётся в функцию. Правила различаются для скалярных и ссылочных типов.

### Скалярные типы (`int`, `uint`, `real`, `bool`)

Скаляры **всегда передаются по значению** — функция получает копию. Модификатор влияет только на локальное переприсвоение внутри функции.

| Синтаксис | Можно переприсвоить локально | Влияние на caller |
|---|---|---|
| `int x` | Нет | Нет |
| `val int x` | Нет | Нет |
| `var int x` | Да | Нет |

```wes
real <- clamp(var real x, real lo, real hi)
{
    run (x < lo) { x = lo }   # OK: var разрешает локальное переприсвоение
    run (x > hi) { x = hi }
    return x
}
```

### Массивы (`array[T]`)

Массивы — ссылочные типы. Модификатор управляет правами доступа к содержимому.

| Синтаксис | Чтение элементов | Запись элементов | Переприсвоение переменной | Изменения видны caller |
|---|---|---|---|---|
| `array[T] arr` | Да | Да | Нет | Да |
| `val array[T] arr` | Да | **Нет** | Нет | — |
| `var array[T] arr` | Да | Да | Да | Да |

```wes
# val: компилятор запрещает любую запись в массив
int <- sum(val array[int] nums)
{
    var int total = 0
    repeat (i in 0 .. nums.size)
    {
        total = total + nums[i]   # OK: чтение
    }
    # nums[0] = 0   # ОШИБКА: val — нельзя менять элементы
    return total
}

# var: изменения видны вызывающей стороне
void <- fill(var array[int] arr, int value)
{
    repeat (i in 0 .. arr.size)
    {
        arr[i] = value   # OK: var — изменения видны caller
    }
}
```

Если модификатор не указан, поведение совпадает с `var` (обратная совместимость).

## Возвращаемые значения

Функция возвращает значение через `return`. Тип выражения должен точно совпадать с объявленным типом возврата.

```wes
array[int] <- mapSquare(val array[int] src)
{
    var array[int] result = array[int](src.size)
    repeat (i in 0 .. src.size)
    {
        result[i] = src[i] * src[i]
    }
    return result
}
```

Функции можно вызывать как часть выражения — результат используется напрямую:

```wes
val int total = sum(filter(data, 5))
```

## Рекурсия

Функции могут вызывать сами себя. Параметры при каждом вызове — отдельные копии для скалярных типов, новые ссылки для массивов.

```wes
int <- gcd(var int a, var int b)
{
    run (b >< 0)
    {
        val int tmp = b
        b = a % b
        a = tmp
    }
    return a
}

void <- quickSort(var array[int] arr, int lo, int hi)
{
    when
    {
        lo < hi ->
        {
            val int pi = partition(arr, lo, hi)
            quickSort(arr, lo, pi - 1)
            quickSort(arr, pi + 1, hi)
        }
    }
}
```

## Вызов функций в цепочке

Результат одной функции можно сразу передать в другую. Промежуточные значения не требуют именования.

```wes
println(sum(filter(data, 5)))
```

---

## Правило синтаксиса

Объявления функций находятся на уровне файла — вложенные функции не поддерживаются. Порядок объявления не важен: функцию можно вызвать до её определения в файле.

```wes
void <- main()
{
    println(square(7))   # OK: square определена ниже
}

int <- square(int x)
{
    return x * x
}
```
