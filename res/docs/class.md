# Классы

## Назначение

Классы являются основным механизмом описания пользовательских структур данных в языке.

Класс объединяет данные и операции над ними в единую сущность и позволяет моделировать абстракции предметной области, такие как графы, деревья, матрицы, очереди и другие структуры данных.

Основной целью классов является упрощение разработки алгоритмов за счет инкапсуляции внутреннего устройства структур данных и предоставления удобного интерфейса для работы с ними.

Классы не предназначены для управления памятью, реализации сложной объектной иерархии или метапрограммирования.

---

# Объявление класса

Класс объявляется с помощью ключевого слова `class`.

Пример:

```text
class Matrix
{
    var uint rows
    var uint cols
    var array[int] data
}
```

Класс может содержать:

* поля;
* конструкторы;
* методы.

---

# Поля

Поля определяют состояние объекта.

Пример:

```text
class Matrix
{
    var uint rows
    var uint cols
    var array[int] data
}
```

Доступ к полям осуществляется через оператор точки.

```text
matrix.rows
matrix.cols
```

Поля могут иметь любой допустимый тип языка.

---

# Создание объектов

Объекты создаются вызовом конструктора.

Пример:

```text
var Matrix matrix = Matrix(10, 20)
```

Создание объекта не требует использования специальных операторов выделения памяти.

В языке отсутствуют конструкции:

```text
new
delete
malloc
free
```

Управление памятью полностью выполняется средой исполнения.

---

# Конструкторы

Конструктор представляет собой специальный метод, имя которого совпадает с именем класса.

Конструктор вызывается автоматически при создании объекта.

Пример:

```text
class Matrix
{
    var uint rows
    var uint cols

    Matrix(uint rows, uint cols)
    {
        this.rows = rows
        this.cols = cols
    }
}
```

Допускается наличие нескольких конструкторов с различными наборами параметров.

Пример:

```text
Matrix()
{
}

Matrix(uint rows, uint cols)
{
}
```

---

# Методы

Методы определяют поведение объекта.

Пример:

```text
class Matrix
{
    void Fill(int value)
    {
        ...
    }

    uint GetRows()
    {
        return rows
    }
}
```

Методы вызываются через оператор точки.

```text
matrix.Fill(0)

var uint rows = matrix.GetRows()
```

---

# Текущий объект

Внутри метода доступ к полям объекта осуществляется напрямую.

Пример:

```text
uint GetRows()
{
    return rows
}
```

При необходимости может использоваться специальная ссылка на текущий объект:

```text
this.rows
```

Однако в большинстве случаев использование `this` не требуется.

---

# Семантика объектов

Все объекты классов являются ссылочными типами.

Пример:

```text
var Matrix a = Matrix(10, 10)

var Matrix b = a
```

После присваивания переменные `a` и `b` ссылаются на один и тот же объект.

Изменение объекта через одну переменную будет видно через другую переменную.

Пример:

```text
a.rows = 100

print(b.rows)
```

Результат:

```text
100
```

Копирование содержимого объекта автоматически не выполняется.

---

# Передача объектов в функции

При передаче объекта в функцию передается ссылка на объект.

Пример:

```text
void Resize(Matrix matrix)
{
    ...
}
```

Создание полной копии объекта не выполняется.

---

# Удаление объектов

Удаление объектов выполняется автоматически средой исполнения.

Язык не предоставляет средств явного освобождения памяти.

Пользователь не обязан отслеживать время жизни объектов.

---

# Наследование

Текущая версия языка не поддерживает наследование классов.

Не допускаются конструкции:

```text
class A : B
{
}
```

Отказ от наследования позволяет сохранить простоту языка и сосредоточиться на решении алгоритмических задач.

---

# Полиморфизм

Текущая версия языка не поддерживает виртуальные методы, интерфейсы и динамический полиморфизм.

Данные возможности могут быть добавлены в будущих версиях языка.

---

# Сравнение объектов

Оператор равенства для объектов сравнивает ссылки на объекты.

Пример:

```text
var Matrix a = Matrix(10, 10)

var Matrix b = a

a == b
```

Результат:

```text
true
```

Если требуется сравнение содержимого объектов, оно должно быть реализовано отдельным методом класса.

---

# Ограничения текущей версии

Поддерживаются:

* классы;
* поля;
* конструкторы;
* методы;
* ссылочная семантика объектов;
* автоматическое управление памятью.

Не поддерживаются:

* наследование;
* интерфейсы;
* виртуальные методы;
* множественное наследование;
* шаблоны;
* пользовательские деструкторы.

---

# Пример

```text
class Matrix
{
    var uint rows
    var uint cols
    var array[int] data

    Matrix(uint rows, uint cols)
    {
        this.rows = rows
        this.cols = cols

        data = array[int](rows * cols)
    }

    uint GetRows()
    {
        return rows
    }

    uint GetCols()
    {
        return cols
    }

    void Fill(int value)
    {
        repeat i in 0..data.size() - 1
        {
            data[i] = value
        }
    }
}
```

Класс инкапсулирует внутреннее устройство матрицы и предоставляет набор операций для работы с ней, не раскрывая детали реализации алгоритмам, использующим данный тип данных.
