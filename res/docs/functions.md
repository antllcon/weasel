# Функции

Функция — именованный блок кода с явными типами параметров и возвращаемого значения. Типы проверяются строго на всех границах вызовов: неявных преобразований нет.

---

## Объявление

Сначала тип возврата, затем стрелка `<-`, имя, параметры в скобках и тело:

```wes
int <- add(int a, int b)
{
    return a + b
}

void <- greet(string name)
{
    println(name)
}
```

Функции без результата объявляются с типом `void`. Они могут завершаться без `return` или содержать `return` без выражения.

Объявления находятся на уровне файла — вложенных функций нет. **Порядок не важен**: функцию можно вызвать выше её определения.

```wes
void <- main()
{
    println(square(7))   # 49 — square определена ниже
}

int <- square(int x)
{
    return x * x
}
```

## Параметры: модификаторы передачи

Каждый параметр может нести модификатор `val` или `var`, управляющий передачей. Правила разные для скаляров и ссылочных типов.

### Скаляры (`int`, `uint`, `real`, `bool`)

Скаляры **всегда передаются по значению** — функция получает копию. Модификатор влияет лишь на локальное переприсвоение.

| Синтаксис | Переприсвоение внутри | Видно caller |
|---|---|---|
| `int x` | нет | нет |
| `val int x` | нет | нет |
| `var int x` | да | нет |

```wes
int <- clamp(var int x, int lo, int hi)
{
    run (x < lo)
    {
        x = lo       # OK: var разрешает локальное переприсвоение
    }
    run (x > hi)
    {
        x = hi
    }
    return x
}
```

### Массивы (`array[T]`)

Массивы — ссылочные типы. Модификатор управляет правами на содержимое.

| Синтаксис | Чтение | Запись | Переприсвоение | Видно caller |
|---|---|---|---|---|
| `array[T] arr` | да | да | нет | да |
| `val array[T] arr` | да | **нет** | нет | — |
| `var array[T] arr` | да | да | да | да |

```wes
# val: компилятор запрещает любую запись в массив
int <- sum(val array[int] nums)
{
    var int total = 0
    repeat (i in 0 .. nums.size)
    {
        total = total + nums[i]   # OK: чтение
    }
    # nums[0] = 0                 # ОШИБКА: val — нельзя менять элементы
    return total
}

# var: изменения видны вызывающей стороне
void <- fill(var array[int] arr, int value)
{
    repeat (i in 0 .. arr.size)
    {
        arr[i] = value
    }
}
```

Без модификатора поведение совпадает с `var`.

## Значения по умолчанию

Параметр может иметь значение по умолчанию — тогда при вызове аргумент можно опустить.

```wes
int <- inc(int x, int by = 1)
{
    return x + by
}

inc(5)      # 6 — by = 1 по умолчанию
inc(5, 3)   # 8 — by = 3
```

Если умолчания заданы у всех параметров, функцию можно вызвать вообще без аргументов:

```wes
int <- box(int w = 10, int h = 20)
{
    return w * h
}

box()        # 200
box(5)       # 100
box(5, 5)    # 25
```

Значение по умолчанию — выражение из литералов, глобальных констант и арифметики. Оно **не видит** другие параметры и вычисляется в момент вызова:

```wes
int <- offset(int base, int delta = 3 + 4)   # delta = 7
```

Параметры с умолчанием обязаны идти **после** обязательных — «дыры» запрещены:

```wes
int <- bad(int a = 1, int b)   # ОШИБКА: обязательный b после дефолтного a
```

> Тип значения по умолчанию проверяется против типа параметра по тем же строгим правилам, что и обычный аргумент.

## Перегрузка

Несколько функций могут носить одно имя, если различаются **сигнатурой** — числом или типами параметров. Подходящую выбирает компилятор по аргументам вызова.

```wes
int <- area(int side)      { return side * side }
int <- area(int w, int h)  { return w * h }

string <- describe(int x)   { return "целое" }
string <- describe(real x)  { return "вещественное" }
string <- describe(bool x)  { return "логическое" }

area(5)          # 25            -> area(int)
area(3, 4)       # 12            -> area(int, int)
describe(42)     # целое         -> describe(int)
describe(3.14)   # вещественное  -> describe(real)
describe(true)   # логическое    -> describe(bool)
```

Перегрузки различаются только по **типам** параметров. Модификаторы `val`/`var` сигнатуру не меняют:

```wes
int <- f(int x)      { return x }
int <- f(var int x)  { return x }   # ОШИБКА: та же сигнатура f(int)
```

> Язык строгий и не приводит аргументы между перегрузками. Если под типы аргументов не подходит ни одна перегрузка — ошибка «нет подходящей перегрузки»; передавайте аргументы точного типа.

## Возвращаемое значение обязательно

Функция с непустым типом возврата обязана вернуть значение на **всех** путях выполнения — компилятор это проверяет.

```wes
int <- sign(int x)
{
    if (x > 0)
    {
        return 1
    }
    # ОШИБКА: не все пути возвращают значение
}
```

Достаточно, чтобы каждая ветвь гарантированно завершалась `return`. Закрыть остаток можно хвостовым `return`:

```wes
int <- sign(int x)
{
    if (x > 0)
    {
        return 1
    }
    else if (x < 0)
    {
        return -1
    }
    return 0
}
```

Развилка `when` с веткой `else`, где все ветви возвращают, тоже считается исчерпывающей:

```wes
real <- abs(real x)
{
    when
    {
        x < 0.0 -> return x * -1.0
        else    -> return x
    }
}
```

Тип выражения в `return` должен точно совпадать с объявленным типом возврата. На `void`-функции правило не распространяется.

## Рекурсия

Функция может вызывать саму себя. Для скаляров каждый вызов получает отдельную копию параметров, для массивов — новую ссылку.

```wes
int <- factorial(int n)
{
    if (n <= 1)
    {
        return 1
    }
    return n * factorial(n - 1)
}

void <- quickSort(var array[int] arr, int lo, int hi)
{
    when
    {
        lo < hi -> {
            val int pi = partition(arr, lo, hi)
            quickSort(arr, lo, pi - 1)
            quickSort(arr, pi + 1, hi)
        }
    }
}
```

## Вызовы в цепочке

Результат одной функции можно сразу передать в другую — промежуточные значения не требуют имён:

```wes
println(sum(filter(data, 5)))
```
