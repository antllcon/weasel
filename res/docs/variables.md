# Переменные и константы

В Weasel каждая переменная объявляется с явным указанием **модификатора** и **типа**. Модификатор определяет, можно ли изменить значение после инициализации, и влияет на то, как компилятор работает с переменной на уровне байткода.

Три модификатора: `val`, `var`, `let`.

---

## var — изменяемая переменная

`var` объявляет переменную, значение которой можно менять в любой момент. Инициализатор необязателен: если значение не указано, переменная получает нулевое значение своего типа (`0`, `0.0`, `false` или пустую ссылку).

```wes
var int counter = 0
var real temperature

counter = counter + 1        # разрешено
temperature = 36.6           # разрешено
```

Мутабельность распространяется на саму переменную, но не на содержимое ссылочных типов. Для массива, объявленного как `var`, можно заменить ссылку целиком и изменять элементы:

```wes
var array[int] items = [1, 2, 3]
items = [4, 5, 6]            # переприсвоение ссылки — разрешено
items[0] = 99                # изменение элемента — разрешено
```

---

## val — неизменяемая переменная

`val` объявляет переменную, которой можно присвоить значение ровно один раз — при объявлении. Попытка переприсвоения — ошибка компиляции.

```wes
val int maxScore = 1000
val string greeting = "Hello, Weasel!"

# maxScore = 999             # Ошибка: нельзя присвоить значение неизменяемой переменной
```

Инициализатор обязателен: объявить `val` без значения нельзя.

Для массивов `val` защищает **ссылку**, но не содержимое. Заменить сам массив нельзя, однако изменять элементы можно:

```wes
val array[int] scores = [10, 20, 30]
scores[0] = 99               # разрешено — изменяем элемент
# scores = [1, 2, 3]        # Ошибка — переприсвоение ссылки
```

| | `var` | `val` |
|---|---|---|
| Переприсвоение значения | Разрешено | Запрещено |
| Переприсвоение ссылки на массив | Разрешено | Запрещено |
| Изменение элементов массива | Разрешено | Разрешено |
| Инициализатор обязателен | Нет | Да |

---

## let — константа времени компиляции

`let` объявляет константу, значение которой **вычисляется и встраивается компилятором** — без выделения стекового слота во время выполнения. Каждое обращение к такой константе заменяется непосредственным вычислением её выражения.

```wes
let int maxRetries = 5
let real pi = 3.14159265
let string appName = "Weasel App"
```

`let` применяется там, где значение принципиально постоянно и известно на этапе компиляции. Поскольку стек-слот не выделяется, это быстрее `val` — особенно для значений, используемых в горячих путях.

```wes
let int bufferSize = 1024

repeat (int i = 0; i < bufferSize; i = i + 1)
{
    # bufferSize здесь не LoadLocal, а встроенная константа 1024
}
```

Инициализатор обязателен и должен быть выражением, не содержащим побочных эффектов.

## Области видимости

Переменные живут в той области видимости, в которой объявлены, и уничтожаются при её завершении. Вложенные области могут объявлять переменные с теми же именами — они скрывают, но не уничтожают внешние.

```wes
val int x = 10
{
    val int x = 20    # отдельная переменная, скрывает внешнюю x
    println(x)        # 20
}
println(x)            # 10 — внешняя x снова видна
```

Переменная цикла, объявленная в заголовке `repeat`, видна только внутри тела:

```wes
repeat (int i = 0; i < 5; i = i + 1)
{
    println(i)        # i доступна
}
# println(i)          # Ошибка: i не видна здесь
```

## Выбор модификатора

| Ситуация | Модификатор |
|---|---|
| Счётчик, аккумулятор, изменяемое состояние | `var` |
| Результат вычисления, который не должен меняться | `val` |
| Магическое число, размер буфера, именованная константа | `let` |
| Параметры функции (внутри тела) | объявляются компилятором как `val` |
