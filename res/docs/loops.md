# Циклы

Weasel предоставляет три вида циклических конструкций, каждая из которых оптимизирована под конкретный сценарий. Все они безопасны: условие всегда должно иметь тип `bool`, а переменная цикла живёт ровно столько, сколько нужно, и автоматически исчезает по его завершении.

---

## run — цикл с предусловием

Цикл `run` выполняет тело, пока условие истинно. Условие проверяется **перед** каждой итерацией, поэтому если с самого начала условие `false`, тело не выполнится ни разу.

```wes
var int i = 0
run (i < 5)
{
    print(i)
    i = i + 1
}
```

Условие обязано быть выражением типа `bool`. Компилятор откажется компилировать код, в котором в условие передаётся `int`, `real` или любой другой тип — в Weasel нет понятия "truthy".

```wes
var bool active = true
var int count = 0
run (active)
{
    count = count + 1
    active = count < 10
}
```

## do-run — цикл с постусловием

Конструкция `do-run` гарантирует, что тело выполнится **хотя бы один раз**: сначала выполняется тело, потом проверяется условие.

```wes
var int x = 100

{
    println(x)
    x = x + 1
} run (x < 5)

# Выведет 100, хотя 100 >= 5: тело выполнилось до проверки
```

Эта форма особенно удобна для обработки пользовательского ввода, первого шага инициализации или любой логики по принципу «сделай, потом проверь».

## repeat — цикл по диапазону

Для перебора диапазона целых чисел используется `repeat` с оператором `in` и разделителем `..`. Переменная-итератор объявляется внутри заголовка и недоступна снаружи.

```wes
repeat (i in 0 .. 10)
{
    print(i)
}
```

Тип итератора выводится из выражений-границ. Если границы имеют тип `uint` — итератор становится `uint`, если `int` — `int`. Типы обеих границ должны совпадать.

```wes
val uint limit = data.size
repeat (i in 0 .. limit)
{
    print(data[i])
}
```

Правая граница **не включается** в диапазон — диапазон `0 .. 5` даёт значения `0, 1, 2, 3, 4`.

Границы могут быть произвольными выражениями:

```wes
repeat (i in base .. base + count)
{
    print(i)
}
```

### Двумерный диапазон

Для одновременного перебора двух диапазонов используется расширенная форма с двумя итераторами и двумя диапазонами через запятую. Это даёт прямую поддержку обхода матриц и двумерных структур.

```wes
repeat (i, j in 0 .. rows, 0 .. cols)
{
    print(matrix[i][j])
}
```

## repeat — классический for

Классическая форма цикла `repeat` с тремя блоками даёт полный контроль над инициализацией, условием и шагом. Это тот же цикл `for`, знакомый из большинства языков программирования.

```wes
repeat (int i = 0; i < 10; i = i + 1)
{
    print(i)
}
```

**Три блока:**
- `int i = 0` — объявление переменной цикла с начальным значением
- `i < 10` — условие продолжения (тип `bool`, проверяется перед каждой итерацией)
- `i = i + 1` — шаг, выполняется после каждой итерации тела

Переменная цикла видна только внутри его тела. В качестве типа переменной можно использовать любой базовый тип (`int`, `uint`, `real`, `bool`, `string`).

Шаг может быть любым присвоением, а не только инкрементом:

```wes
# Обратный отсчёт с шагом 2
repeat (int i = 10; i > 0; i = i - 2)
{
    print(i)
}
```

## Вложенные циклы

Любые виды циклов можно свободно вкладывать друг в друга. Переменные цикла из разных уровней не конфликтуют.

```wes
repeat (int i = 1; i <= 3; i = i + 1)
{
    repeat (int j = 1; j <= 3; j = j + 1)
    {
        print(i * j, " ")
    }
    println("")
}
```

---

## Выбор подходящего цикла

| Ситуация | Цикл |
|---|---|
| Количество итераций известно, перебор диапазона | `repeat (i in 0 .. n)` |
| Счётчик с нестандартным шагом или условием | `repeat (init; cond; step)` |
| Условие проверяется перед входом | `run (cond)` |
| Тело должно выполниться хотя бы раз | `do { } run (cond)` |
