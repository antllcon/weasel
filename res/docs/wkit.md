# wkit — инструмент управления проектами Weasel

`wkit` создаёт, собирает, запускает и форматирует проекты на Weasel. Он работает поверх компилятора (вызывает его напрямую), поэтому отдельно запускать `wesc` не нужно.

---

## Установка (добавление в PATH)

`wkit.exe` лежит в каталоге `res` рядом с `grammar.wesg` и (опционально) `stdlib`. **Грамматику и stdlib `wkit` ищет рядом с собой**, поэтому `wkit.exe` нельзя переносить отдельно от `res` — вместо этого добавьте сам каталог `res` в `PATH`.

**Windows 11 — через параметры:**
1. «Пуск» → «Изменение переменных среды текущего пользователя».
2. В разделе «Переменные среды пользователя» выберите `Path` → «Изменить» → «Создать».
3. Вставьте путь к каталогу (например `C:\dev\weasel\res`) → «ОК».
4. Откройте **новый** терминал — изменения применяются только в новых окнах.

**Windows 11 — через PowerShell (для текущего пользователя):**
```powershell
[Environment]::SetEnvironmentVariable(
    "Path",
    [Environment]::GetEnvironmentVariable("Path", "User") + ";C:\dev\weasel\res",
    "User")
```
После этого `wkit` доступен из любого каталога. Проверка: `wkit` без аргументов покажет список команд.

## Команды

| Команда | Действие |
|---|---|
| `wkit new <name>` | Создать новый проект в каталоге `<name>`. |
| `wkit build` | Скомпилировать и проверить проект **без запуска**. |
| `wkit run` | Скомпилировать и запустить проект. |
| `wkit debug` | Запустить проект с **подробным логом компиляции** (токены, CST, AST, таблица символов) — как флаг `-l` у `wesc`. |
| `wkit format` | Отформатировать исходники проекта (`.wes`, кроме `wkit.wes`). |

`build`/`run`/`format` запускаются **из каталога проекта** (там, где лежит `wkit.wes`).

## Структура проекта

```
demo/
├── wkit.wes        # конфигурация проекта
└── src/
    └── main.wes    # точка входа
```

`wkit new demo` создаёт именно такую структуру с готовым «Привет, Weasel!».

---

## Конфигурация `wkit.wes`

Конфиг написан **на самом языке Weasel** — это набор констант, которые читает `wkit`:

```wes
let string name = "demo"
let string entry = "src/main.wes"
# Необязательно: свой каталог stdlib (по умолчанию — рядом с wkit)
# let string stdlib = "stdlib"
```

| Ключ | Значение |
|---|---|
| `name` | Имя проекта (информационно). |
| `entry` | Главный файл программы (относительно каталога проекта). |
| `stdlib` | Необязательный каталог стандартной библиотеки. Если не задан — берётся каталог `stdlib` рядом с `wkit`. |

Пути `import` внутри программы разрешаются как обычно: сначала рядом с файлом, затем в каталоге stdlib (см. `modules.md`).

---

## Пример

```
wkit new demo
cd demo
wkit run        # Привет, Weasel!
```

`wkit build` ничего не выводит при успехе (кроме `wkit: сборка успешна`) и завершается с ненулевым кодом, если есть ошибки компиляции — удобно для скриптов и CI. Текст ошибок печатается всегда (см. `cli.md`).
