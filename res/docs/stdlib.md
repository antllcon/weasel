# Стандартная библиотека: контейнеры

Стандартная библиотека Weasel написана **на самом языке Weasel** и подключается через `import` (см. `modules.md`). Контейнеры лежат в каталоге stdlib (по умолчанию — рядом с `wkit`/`wesc`).

В текущей версии контейнеры работают с фиксированными типами элементов (целые числа). Все они хранят данные в `array[…]` и растут автоматически — ёмкость увеличивается **в 1.5 раза** (целочисленно, `capacity + capacity / 2`): это меньший перерасход памяти и лучшее переиспользование освобождённых блоков аллокатором, чем рост ×2.

Если итоговый размер известен заранее, аллокаций при заполнении можно избежать: задайте ёмкость в конструкторе (`IntList(100)`) или вызовите `reserve(n)`.

```wes
import "list"
import "stack"
import "queue"
import "map"
```

---

## `IntList` — динамический список (`list`)

Список целых чисел переменной длины (аналог `vector<int>`). Растёт в 2 раза при заполнении.

| Метод | Действие |
|---|---|
| `IntList()` | Создать пустой список. |
| `IntList(initialCapacity)` | Создать пустой список с зарезервированной ёмкостью. |
| `size()` | Число элементов. |
| `isEmpty()` | `true`, если список пуст. |
| `reserve(minCapacity)` | Заранее выделить ёмкость не меньше `minCapacity`. |
| `push(value)` | Добавить элемент в конец. |
| `get(index)` | Элемент по индексу. |
| `set(index, value)` | Заменить элемент по индексу. |
| `pop()` | Удалить и вернуть последний элемент. |

```wes
var IntList xs = IntList()
xs.push(10)
xs.push(20)
println(xs.get(0))   # 10
println(xs.pop())    # 20
println(xs.size())   # 1
```

## `IntStack` — стек (`stack`)

Стек целых чисел, LIFO (последним пришёл — первым вышел).

| Метод | Действие |
|---|---|
| `IntStack()` | Создать пустой стек. |
| `IntStack(initialCapacity)` | Создать стек с зарезервированной ёмкостью. |
| `size()` / `isEmpty()` | Размер / проверка на пустоту. |
| `push(value)` | Положить на вершину. |
| `pop()` | Снять и вернуть вершину. |
| `peek()` | Посмотреть вершину, не снимая. |

---

## `IntQueue` — очередь (`queue`)

Очередь целых чисел, FIFO (первым пришёл — первым вышел). Реализована как **кольцевой буфер**: `enqueue`/`dequeue` работают за O(1) без копирования (индексы по модулю ёмкости); копирование происходит только при росте.

| Метод | Действие |
|---|---|
| `IntQueue()` | Создать пустую очередь. |
| `IntQueue(initialCapacity)` | Создать очередь с зарезервированной ёмкостью. |
| `size()` / `isEmpty()` | Размер / проверка на пустоту. |
| `enqueue(value)` | Добавить в хвост. |
| `dequeue()` | Извлечь и вернуть из начала. |
| `front()` | Посмотреть начало, не извлекая. |

---

## `Map` — ассоциативный массив (`map`)

Отображение `string -> int`. Реализация — линейный поиск (просто и наглядно, сложность O(n)).

| Метод | Действие |
|---|---|
| `Map()` | Создать пустое отображение. |
| `size()` | Число пар. |
| `set(key, value)` | Вставить или обновить значение по ключу. |
| `get(key)` | Значение по ключу (или `0`, если ключа нет). |
| `contains(key)` | `true`, если ключ есть. |

```wes
var Map ages = Map()
ages.set("Alice", 30)
ages.set("Alice", 31)        # обновление
println(ages.get("Alice"))   # 31
println(ages.contains("Eve")) # false
```

---

## Замечания

- **Проверка границ.** `get`/`set`/`pop`/`peek`/`dequeue`/`front` не проверяют пустоту и границы — это ответственность вызывающего (в языке нет исключений, см. `cli.md`). Выход за границы массива приводит к аварийной остановке VM.
- **Только целые элементы.** `list`/`stack`/`queue` хранят `int`, `map` — ключи `string`, значения `int`. Контейнеры для произвольных типов требуют дженериков (планируются отдельно).
- **Ссылочная семантика.** Контейнеры — это классы, то есть передаются по ссылке: передача контейнера в функцию не копирует его.
