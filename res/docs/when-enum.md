# Сопоставление с образцом: when / enum

В Weasel есть два инструмента ветвления под разные задачи. Конструкция `if` подходит для одного-двух условий. Когда вариантов становится больше — в игру вступает `when`: лаконичная и выразительная форма сопоставления с образцом, вдохновлённая современными языками.

`when` является конструкцией управления потоком (statement) и не возвращает значение напрямую. Как и везде в Weasel, все условия строго типизированы — неявных преобразований к `bool` нет.

Как и во всем языке, в ветвлениях действует правило строгой статической типизации: любое логическое условие обязано вычисляться строго в тип `bool`. В Weasel нет скрытых приведений чисел или пустых строк к истине или лжи.

### Классическое ветвление (`when` без предмета)

Если вам нужно последовательно проверить ряд независимых логических условий, используйте `when` без параметров. Вычисление идет сверху вниз ленивым образом. Как только виртуальная машина находит истинное условие, выполняется соответствующая ветка (после оператора `->`), а остальные игнорируются.

Для обработки ситуации, когда ни одно из условий не подошло, используется ключевое слово `else`.

```wes
val int score = 85

when 
{
    score >= 90 -> print("Оценка: Отлично")
    score >= 70 -> print("Оценка: Хорошо")
    score >= 50 -> print("Оценка: Удовлетворительно")
    else -> print("Оценка: Неуд")
}

```

### Сопоставление с предметом

Часто логика требует сравнения одной и той же переменной с разными значениями. Для этого в `when` можно передать "предмет" (subject) в круглых скобках. Это элегантная, читаемая и безопасная альтернатива классическому оператору `switch`.

Ветки могут проверять как одно значение, так и сразу несколько — достаточно перечислить их через запятую. Типы проверяемых значений обязаны строго совпадать с типом переданного предмета.

```wes
val int dayCode = 4

when (dayCode) 
{
    1, 2, 3, 4, 5 -> print("Это будний день. Надо работать.")
    6, 7 -> print("Это выходной? Нет, это six-seven")
    else -> print("Ошибка: неизвестный день недели")
}

```

### Перечисления (`enum`) и исчерпываемость

Одно из главных преимуществ оператора `when` раскрывается при работе с пользовательскими перечислениями (`enum`). Компилятор Weasel осуществляет семантическую проверку ветвлений на **исчерпываемость** (exhaustiveness).

Если вы передаете переменную типа `enum` в качестве предмета и намеренно не пишете ветку `else`, компилятор на этапе сборки проверит, что вы обработали абсолютно все возможные значения этого перечисления.

Если в будущем вы добавите новый статус в `enum`, но забудете обновить `when`, компилятор выдаст ошибку и программа не скомпилируется. Это полностью исключает целый класс багов, связанных с забытыми состояниями приложения.

```wes
enum Status 
[ 
    NEW, 
    PROCESSING, 
    DONE, 
    ERROR 
]

val Status currentTask = Status.PROCESSING

# Ветка else здесь не нужна, компилятор гарантирует покрытие всех полей
when (currentTask) 
{
    Status.NEW -> print("Задача только создана")
    Status.PROCESSING -> print("Задача в процессе выполнения...")
    Status.DONE -> print("Задача успешно завершена")
    Status.ERROR -> print("Произошла ошибка при выполнении")
}

```

### Блоки кода в ветках

Логика внутри ветки не ограничивается одним выражением. Если при выполнении условия требуется сделать несколько действий, объявить локальные переменные или вызвать несколько функций, просто оберните тело ветки в фигурные скобки `{ ... }`.

```wes
val uint code = 200

when (code) 
{
    200, 201 -> {
        print("Успех!")
        val uint extraData = 42
        print("Дополнительные данные:", extraData)
    }
    else -> print("Произошла ошибка")
}

```

Открывающую скобку можно ставить как сразу за `->`, так и на новой строке — обе формы равноценны:

```wes
when (code)
{
    200 ->
    {
        print("Успех!")
    }
    else -> print("Ошибка")
}
```

> Сам `when` пишите многострочно — каждая ветвь на своей строке. Тело ветви идёт сразу за `->` (или с новой строки для блока).
