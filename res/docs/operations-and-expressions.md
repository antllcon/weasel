# Операции и Выражения

Выражения строятся по классическим математическим правилам с учетом строгой статической типизации. Компилятор никогда не позволит вам случайно сложить строку с числом или потерять точность при неявном преобразовании дроби в целое.

### Арифметические операции

Язык поддерживает стандартный набор арифметических операторов для всех числовых типов (`int`, `uint`, `real`):

- `+` (сложение)
- `-` (вычитание)
- `*` (умножение)
- `/` (деление)
- `%` (остаток от деления)

Операции деления `/` и взятия остатка `%` полностью безопасны. Попытка деления на ноль или переполнение (например, деление минимального `int` на `-1`) не приведет к падению приложения на уровне ОС, а будет аккуратно перехвачена виртуальной машиной.

### Сравнение и Логика

Все операции сравнения и логики всегда возвращают тип `bool`.

Для сравнения значений используются операторы: `==`, `><`, `<`, `>`, `<=`, `>=`.

Для построения сложных условий используются текстовые логические операторы. Они обладают пониженным приоритетом по сравнению с арифметикой, что позволяет писать условия без лишних скобок:

- `and` — логическое И
- `or` — логическое ИЛИ
- `not` — логическое отрицание

``` wes
val int age = 25
# Скобки не обязательны, приоритет операторов выстроен естественно
val bool isAdult = age >= 18 and not (age == 100)
val bool mismatch = age >< 30
```

**Короткое замыкание**: Вычисление логических выражений `and` и `or` происходит лениво (слева направо). Если при вычислении or левая часть равна `true`, правая часть даже не будет вычисляться. Это позволяет безопасно писать конструкции вроде проверки на `null` перед вызовом.

### Работа со строками

Операторы `+`, `==` и `><` перегружены для работы со строковым типом данных `string`. Сложение строк всегда порождает новую строку в динамической памяти, а операции сравнения честно проверяют содержимое (текст), а не адреса в памяти.

```wes
val string hello = "Hello"
val string world = "World"
val string result = hello + " " + world 

val bool isEqual = hello == "Hello"
```

### Что можно и нельзя

Из-за строгой типизации в языке существуют четкие правила взаимодействия типов внутри выражений.

Что в языке делать можно?
1. Смешивать переменные и числовые литералы. 
Компилятор Weasel обладает умным выводом типов. Числовые литералы (например, 10, 42) не имеют жестко зафиксированного типа до момента компиляции. Они автоматически "подстраиваются" под тип переменной, с которой взаимодействуют.

```wes
val uint base = 100
# Литералы 10 и 20 автоматически станут uint без ошибок
val uint infer = 10 + 20 + base
```

2. Использовать цепочки унарных операторов 
Вы можете безопасно комбинировать унарные операции, если это требуется математической логикой.

```wes
val int a = - - -42               # Результат: -42
val bool b = not not not true     # Результат: false
```
Что в языке делать нельзя?

1. Неявно смешивать разные базовые типы.
Компилятор запретит любую математику или сравнение между int и real, int и uint без явного приведения типов. Это защищает от потери знака или дробной части.

```wes
val int a = 10
val real b = 3.14
# val error1 = a + b              # Ошибка: строгое несоответствие типов
val real ok = a.real() + b        # Пример явного каста
```

2. Складывать строки с числами напрямую.
В отличие от некоторых скриптовых языков, Weasel не преобразует числа в строки автоматически при конкатенации.

```wes
val int count = 5
# val error2 = "Items: " + count  # Ошибка
```

3. Использовать не-булевы типы в логических условиях.
В Weasel нет понятия "truthy" или "falsy" значений (как в C++ или JavaScript). Число `0` — это число, а не `false`. Пустая строка `""` — это строка. Логические операторы (`and`, `or`, `not`), а также условия ветвления принимают строго тип `bool`.

```wes
val int value = 1
# val error = not value           # Ошибка: ожидался bool
val bool ok = not (value == 1)    # Правильно
```
